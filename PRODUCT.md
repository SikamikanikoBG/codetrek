# Product

## Register

product

## Users

Two children (mixed age — from pre-reader up through fluent reader) learning programming fundamentals at home, on both desktop and mobile/touch browsers. No login: each kid picks their own local profile (name + avatar) and plays independently. The same app must serve a pre-reader who can't read a block label yet and an older sibling who's ready for loops, conditionals, and eventually real Python — without feeling like two different products. Longer term, this is a public open-source project other families may adopt.

## Product Purpose

CodeTrek teaches programming fundamentals (sequencing → loops → conditionals → variables → eventually Python and AI-assistant fluency) through a Blockly-powered visual editor driving a robot-on-a-grid puzzle game. Progress is gamified (XP, stars, badges) and gated by demonstrated skill, not fixed age. Success looks like: a kid opens the app unprompted, plays because it's fun, and picks up real programming concepts as a side effect of solving puzzles.

## Brand Personality

**Scratch-like playful craft.** Candy-bright but deliberate color use, friendly rounded shapes, hand-crafted warmth — the feeling of a well-made toy or a craft tool for kids, not a stock template. Confident and inviting rather than loud; a kid should feel like they're building something, not just clicking through a game.

## Anti-references

- **Cheap clip-art/cartoon look**: no generic stock cartoon characters, no gradient text, no amateurish clip-art — it must feel crafted, not templated.
- **Sterile corporate edtech**: no bland school-SaaS dashboard feeling — no gray cards, boring form-heavy screens, or anything that reads as "homework software."

## Design Principles

1. **Show, don't tell.** The icon-only tier carries zero required text — visuals, color, and motion do the explaining before literacy can.
2. **Craft over cartoon.** Playful is not the same as cheap. Every screen gets the attention of a well-made physical toy, not a generic template.
3. **One clear next action.** A kid should never be lost — there is always one obvious next thing to try (which block to grab, which level to enter).
4. **Celebrate small wins loudly.** XP, stars, and badges are the emotional payoff of the whole product — the feedback must feel immediate and satisfying, not perfunctory.
5. **One app, two paces.** The interface flexes from wordless icon-tier to text-labeled block-tier without ever feeling like a different product underneath.

## Accessibility & Inclusion

WCAG AA contrast minimum. Colorblind-safe palette — status (locked/unlocked/completed, correct/incorrect) must never rely on color alone; pair with shape/icon/position. Full `prefers-reduced-motion` support for all celebratory/motion moments.
